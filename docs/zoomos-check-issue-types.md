# Zoomos Check — Типы проблем (Issue Types)

## Основные типы (поле `type` в issues)

| Тип | Цвет | Блокирует canDeliver | Когда срабатывает |
|-----|------|-----------------------|-------------------|
| **ERROR** | Красный `#dc3545` | **Да** | Падение "В наличии" > порога, нет данных по городу/адресу, парсер-фильтр не завершён |
| **WARNING** | Жёлтый `#ffc107` | Нет | 100% выкачка но 0 товаров, всегда 0 в наличии, рост ошибок, выкачка незавершена |
| **TREND** | Голубой `#0d9aad` | Нет | Исторический анализ: падение доли in-stock, рост ошибок, замедление скорости |

CSS классы: `issue-error`, `issue-warning`, `issue-notfound`, `issue-trend` (check-results.html:19-22)

---

## Логические состояния в UI (реализованы через WARNING/ERROR)

| Состояние | Реальный тип | Условие |
|-----------|-------------|---------|
| **IN_PROGRESS** | WARNING | Выкачка ещё идёт |
| **Зависла** | WARNING | `updatedTime` не обновлялся > 2 часов |
| **Нет данных (NO_DATA)** | ERROR | Вообще нет никаких записей |
| **NOT_FOUND** | ERROR | Нет данных по конкретному адресу/городу |

---

## Подтипы ERROR

1. **Парсер-фильтр не завершён** — `"Ни один парсер '[pattern]' не завершён"` (OR) / `"Парсер '[pattern]' не завершён"` (AND)
2. **Падение "В наличии"** — `"Падение 'В наличии': [медиана: N] → M (−X%)"` (> dropThreshold, по умолчанию 10%)
3. **"В наличии" упал на 100%** — от ненулевого к нулю
4. **Нет данных по адресу/городу** — `"Нет данных по адресу [addressId] за указанный период"`

## Подтипы WARNING

1. **100% выкачка, но нет товаров** — `"100% выкачка, нет товаров — нужна проверка"`
2. **Первая запись: 0 из N в наличии** — `"В наличии: 0 из [total] товаров (первая запись)"`
3. **Всегда 0 в наличии** — `"В наличии: всегда 0 — нужна проверка"`
4. **Рост ошибок** — >errorGrowthThreshold (по умолчанию 30%)

## Подтипы TREND (блок 3, исторический анализ)

1. **Падение доли "В наличии"** — `"Доля 'В наличии' снизилась: было X% → стало Y% (−Z%)"`
2. **Рост ошибок** — `"Рост ошибок: было X% → стало Y%"` или `"Появились ошибки: X%"`
3. **Замедление выкачки** — `"Скорость снизилась: было X.XX → стало Y.YY мин/1000 тов."`

---

## Счётчики в ZoomosCheckRun

```
ok_count / warning_count / error_count / not_found_count
```

Вычисляются **динамически** из `siteCityStatuses` при рендере, **не из БД**.
