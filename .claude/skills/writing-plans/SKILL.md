# Writing Plans

**ГЛАВНОЕ ПРАВИЛО: Никакого кода до одобрения плана.**

Это применимо даже к "простым" задачам — план всегда дешевле неверной реализации.

---

## Процесс создания плана

### Шаг 1: Изучи контекст

Перед планированием:
- Прочитай `CLAUDE.md` — принципы, стек, паттерны
- Изучи похожий существующий код в проекте
- Проверь есть ли уже реализовано что-то похожее
- Задай **один уточняющий вопрос за раз** если цель неясна

```bash
# Найти похожие реализации
grep -r "ключевое_слово" src/main/java/ --include="*.java" -l
```

### Шаг 2: Предложи 2-3 подхода с трейдоффами

Формат для каждого подхода:
```
Подход N: [название]
+ Плюсы
- Минусы
Когда выбирать: [условие]
```

### Шаг 3: Напиши детальный план

#### Структура плана:

**Цель:** Одна фраза о результате.

**Затронутые файлы:**
- `src/main/java/com/java/service/...` — что изменится
- `src/main/resources/db/migration/V*.sql` — если нужна миграция

**Шаги (каждый ≤5 минут):**

| # | Действие | Файл | Тест |
|---|----------|------|------|
| 1 | Написать failing тест | `...Test.java` | — |
| 2 | Запустить тест → убедиться что RED | — | mvn test -Dtest=... |
| 3 | Реализовать [конкретно что] | `Service.java` | — |
| 4 | Запустить тест → GREEN | — | mvn test -Dtest=... |
| 5 | Проверить компиляцию | — | mvn compile -q |

### Шаг 4: Валидация плана

Перед отправкой проверь:
- [ ] Каждый шаг конкретен (что именно написать/изменить)
- [ ] Есть тесты для ключевой логики
- [ ] Учтены Critical Non-Obvious Behaviors из CLAUDE.md
- [ ] YAGNI — нет лишних абстракций
- [ ] Если нужна Flyway миграция — она в плане

---

## YAGNI чеклист

Задай себе вопросы:
- Это реально нужно прямо сейчас?
- Есть ли уже готовое в проекте?
- Это усложняет существующий код без необходимости?

Если на любой вопрос ответ "да" — убери.

---

## После одобрения плана

Если план одобрен:
1. Реализуй шаг за шагом в порядке плана
2. После каждого шага с тестом — запусти `mvn test`
3. Зафиксируй: `git add <конкретные файлы> && git commit`
4. Обнови документацию в `docs/` если изменилась функциональность
