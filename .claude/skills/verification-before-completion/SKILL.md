# Verification Before Completion

**ПРИНЦИП: Сначала доказательства, потом утверждения. Всегда.**

Пропуск шагов верификации — это не эффективность, это нечестность.

---

## Обязательный протокол перед любым заявлением "готово"

### 5-шаговый gate

1. **Определи** — что именно доказывает что задача выполнена?
2. **Выполни** — запусти проверочную команду прямо сейчас, в этом сообщении
3. **Прочитай** — изучи полный вывод и exit code
4. **Сопоставь** — вывод соответствует заявлению?
5. **Только тогда** — сделай утверждение, приведя конкретный вывод как доказательство

---

## Проверочные команды для Zoomos v4

### Компиляция
```bash
cd /e/workspace/zoomos_v4 && mvn compile -q 2>&1
# EXIT CODE 0 = компилируется. Любое другое = ошибка.
```

### Конкретный тест
```bash
cd /e/workspace/zoomos_v4 && mvn test -Dtest=ИмяТеста -q 2>&1
# Смотреть: "BUILD SUCCESS" и "Tests run: N, Failures: 0, Errors: 0"
```

### Сервер запущен
```bash
netstat -ano | findstr :8081
# Должна быть строка с LISTENING
```

### Изменения файлов
```bash
cd /e/workspace/zoomos_v4 && git diff --name-only
# Убедиться что изменены именно нужные файлы
```

## Опасные паттерны (запрещены)

- "должно работать" — без запуска проверки
- "вероятно исправлено" — без запуска теста
- "BUILD SUCCESS" написать без реального запуска
- сообщить об успехе субагента без независимой проверки
- "линтер прошёл" → значит сборка тоже прошла (нет, это разные вещи)

## Допустимые формулировки после верификации

```
Компиляция: mvn compile -q → EXIT 0 (вывод чистый)
Готово.
```

```
Тест запущен: mvn test -Dtest=WeekNumberUtilsTest
Результат: Tests run: 3, Failures: 0, Errors: 0
Реализация корректна.
```

**Правило:** Если ты не запустил проверочную команду в этом же сообщении — ты не можешь утверждать что она прошла.
