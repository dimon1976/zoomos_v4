# Соглашения по разработке кода

> Основано на техническом видении проекта из [@vision.md](vision.md)

## Общие принципы

**KISS над всем** - каждое решение должно быть максимально простым. Если можно проще, делай проще.

**Переиспользуй, не изобретай** - используй существующие сервисы проекта:
- `FileAnalyzerService` для анализа файлов
- `FileGeneratorService` для создания результатов
- Существующую систему загрузки файлов

**Никакого оверинжиниринга** - это MVP для проверки идеи, не энтерпрайз система.

## Структура кода

### Размещение файлов
```
controller/utils/DataMergerController.java
service/utils/DataMergerService.java
dto/utils/SourceDataRow.java
dto/utils/CollectedDataRow.java
dto/utils/MergedDataRow.java
templates/utils/data-merger.html
```

### Именование
- Классы: `DataMergerController`, `DataMergerService`
- Методы: `mergeData()`, `processFiles()`
- URL: `/utils/data-merger`, `/utils/data-merger/process`

## Java код

### DTO классы
- Используй `@Data` из Lombok
- Только String поля (простота превыше типобезопасности)
- Никаких валидаций, геттеров, сеттеров вручную

### Service классы
- Один сервис `DataMergerService`
- Один главный метод `mergeData()`
- Используй `Map<String, List<String>>` для поиска соответствий
- Синхронная обработка, никаких `@Async`

### Controller классы
- `GET /utils/data-merger` - форма
- `POST /utils/data-merger/process` - обработка
- Возвращай файл напрямую через `ResponseEntity<Resource>`
- Минимальная обработка ошибок

## Обработка данных

### Алгоритм
1. Читай файлы через `FileAnalyzerService`
2. Преобразуй в List и Map
3. Итерируй и ищи соответствия
4. Создавай результат построчно (каждая ссылка = отдельная строка)
5. Генерируй файл через `FileGeneratorService`

### Обработка ошибок
- Только основные случаи: файл не найден, неверная структура, пустые данные
- Простые сообщения пользователю
- Никаких детальных логов

## Frontend

### HTML форма
- Bootstrap 5.3.0 стили
- Два `<input type="file">` для загрузки
- Одна кнопка "Объединить данные"
- Никакого JavaScript (если не критично)

### Thymeleaf шаблон
- Наследуй от существующего layout
- Используй существующие Bootstrap компоненты
- Простая структура без излишеств

❌ Не делай асинхронную обработку
❌ Не добавляй кэширование
❌ Не создавай сложную валидацию
❌ Не делай продвинутый UI
❌ Не добавляй настройки пользователя
❌ Не создавай историю операций

## Критерий готовности

Код готов, если:
✅ Загружаются 2 файла
✅ Происходит объединение по модели аналога
✅ Результат выгружается построчно
✅ Ошибки обрабатываются базово
✅ Интегрируется с существующими сервисами

**Если работает базовый сценарий - код готов. Все остальное потом.**
