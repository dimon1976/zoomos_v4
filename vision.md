# Техническое видение утилиты соединения данных

## Обзор проекта
Простая утилита для объединения исходных данных с собранными ссылками по совпадению моделей аналогов. MVP решение для проверки идеи с максимальной простотой реализации.

## 1. Технологии

### Backend (переиспользование существующего стека)
- **Spring Boot 3.2.12** - используем существующую инфраструктуру
- **Java 17** - уже настроен в проекте
- **Apache POI 5.2.3 + OpenCSV 5.8** - переиспользуем для работы с файлами

### Frontend (минимальная веб-форма)
- **Thymeleaf** - уже используется в проекте
- **Bootstrap 5.3.0** - уже подключен
- **Vanilla JavaScript** - без дополнительных фреймворков

### Файловая система
- **Локальное хранение** - `data/upload/` (существующая структура)

### Переиспользуемые сервисы
- `FileAnalyzerService` - анализ структуры загруженных файлов
- `FileGeneratorService` - генерация результирующих файлов

## 2. Принципы разработки

### Основные принципы
- **KISS (Keep It Simple, Stupid)** - максимальная простота во всем
- **MVP (Minimum Viable Product)** - только базовый функционал для проверки идеи
- **Переиспользование кода** - максимальное использование существующих сервисов
- **Fail Fast** - быстрая валидация входных данных и немедленное сообщение об ошибках

### Технические принципы
- Один контроллер, один сервис, минимум DTO
- Синхронная обработка без очередей и фоновых задач
- Работа только с файловой системой, без базы данных
- Простая веб-форма без сложного UI
- Минимальная обработка ошибок (основные случаи)

### Ограничения MVP
- Без ограничений на количество строк
- Без логов операций (добавим при необходимости)
- Без истории операций
- Без пользовательских настроек
- Без кэширования результатов
- Полагаемся на существующий `FileAnalyzerService` для валидации форматов

## 4. Архитектура проекта

### Слой представления (Controller)
- `DataMergerController` - принимает 2 файла, возвращает результат
- **Методы:**
  - `GET /utils/data-merger` - отображение формы
  - `POST /utils/data-merger/process` - обработка файлов

### Слой бизнес-логики (Service)
- `DataMergerService.mergeData()` - основной алгоритм слияния
- Использует `FileAnalyzerService` для чтения файлов
- Использует `FileGeneratorService` для создания результата

### Алгоритм обработки
1. Загрузка исходных данных в `List<SourceDataRow>`
2. Загрузка собранных данных в `Map<String, List<String>>` (модель → ссылки)
3. Итерация по исходным данным:
   - Поиск соответствий в Map по ключу "модель аналога"
   - Создание `MergedDataRow` с найденными ссылками (построчно)
4. Генерация CSV/Excel файла с результатами
5. Возврат файла пользователю

### Обработка ошибок (минимальная)
- Файл не найден → сообщение пользователю
- Неверная структура файла → сообщение о требуемых колонках
- Пустые файлы → предупреждение
- Общие ошибки → "Попробуйте еще раз"

## 5. Модель данных

### SourceDataRow (исходные данные)
```java
@Data
public class SourceDataRow {
    private String id;              // ID
    private String clientModel;     // Модель клиента
    private String analogModel;     // Модель аналога
    private String coefficient;     // Коэффициент
}
```

### CollectedDataRow (собранные данные)
```java
@Data
public class CollectedDataRow {
    private String analogModel;     // Модель аналога (ключ)
    private String link;           // Ссылка
}
```

### MergedDataRow (результат - построчная выгрузка)
```java
@Data
public class MergedDataRow {
    private String id;              // ID (из исходных)
    private String clientModel;     // Модель клиента (из исходных)
    private String analogModel;     // Модель аналога (из исходных)
    private String coefficient;     // Коэффициент (из исходных)
    private String link;           // Одна ссылка на строку
}
```

### Формат файлов
- **Исходный файл:** ID | Модель клиента | Модель аналога | Коэффициент
- **Собранные данные:** Модель аналога | Ссылка
- **Результат:** ID | Модель клиента | Модель аналога | Коэффициент | Ссылка

### Логика построчной выгрузки
- Если для модели аналога найдено N ссылок → создается N строк в результате
- Если ссылок не найдено → одна строка с пустым полем "link"
- Исходные данные (ID, модель клиента, коэффициент) дублируются в каждой строке

## 6. Сценарии работы

### Основной сценарий (Happy Path)
1. Пользователь заходит на `/utils/data-merger`
2. Загружает файл "Исходные данные" (CSV/Excel)
3. Загружает файл "Собранные данные" (CSV/Excel)
4. Нажимает "Объединить данные"
5. Получает результирующий файл для скачивания

### Сценарий с ошибками
1. Неверный формат файла → "Файл должен содержать колонки: ..."
2. Пустой файл → "Файл пуст или поврежден"
3. Отсутствует обязательная колонка → "Не найдена колонка 'Модель аналога'"

### Сценарий без совпадений
1. В собранных данных нет моделей аналогов из исходных данных
2. Результат содержит все исходные строки с пустым полем "Ссылка"
